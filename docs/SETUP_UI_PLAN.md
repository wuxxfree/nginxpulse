# 初始化配置向导 - 实施清单

## 目标
- 无需启动参数即可运行镜像，进入初始化向导完成配置。
- 仅在“未提供配置文件/未注入环境变量”时显示向导。
- 涉及路径字段需在宿主机（容器内）校验存在性。
- 向导风格与现有界面一致，每步提供“高级”折叠项。
- 初始化模式无需数据库即可启动 API/UI。
- 配置保存后提示重启，重启后生效。

## 后端
- [ ] 配置来源判断：新增“是否需要初始化”的判断函数。
- [ ] 允许无配置启动（默认配置 + 空网站）。
- [ ] 配置接口：
  - [ ] `GET /api/config` 获取当前配置 + readonly/setup 标记。
  - [ ] `POST /api/config/validate` 校验配置（包含路径存在性）。
  - [ ] `POST /api/config/save` 保存配置（原子写入，更新运行时配置）。
- [ ] 统一校验逻辑（必填项、路径、基础参数范围）。
- [ ] `GET /api/status` 增加 `setup_required`/`config_readonly` 字段。
- [ ] 配置保存后提示重启（端口/数据库/访问密钥需重启生效）。

## 前端
- [ ] 初始化向导页面（全屏/遮罩式）。
- [ ] 多步骤流程 + 步骤进度条。
- [ ] 每步“高级”折叠面板。
- [ ] 必填项校验 + 后端校验提示。
- [ ] JSON 预览 + 保存。
- [ ] 仅在 `setup_required=true` 时展示。
